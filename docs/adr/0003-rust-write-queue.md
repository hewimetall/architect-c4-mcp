# ADR 0003: очередь записи

## Статус

Accepted

## Контекст

Несколько MCP-вызовов могут одновременно менять model, ADR и flows. Записи в
TOML должны оставаться атомарными и предсказуемыми.

## Решение

Все `upsert_*` и delete-операции проходят через in-process очередь Rust с одним
writer. Tool call дожидается результата записи.

## Последствия

- Параллельные агенты не пишут в один файл одновременно.
- После рестарта сохраняется только уже записанное в TOML.
- Python-слой остается тонкой FastMCP-оберткой.
